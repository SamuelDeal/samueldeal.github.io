    Sugar

    This is my toy language. The main idea is to provide a low level language, with a modern syntax, and compile-time metaprogramming. It should be run with an interpreter or compiled (easy to achieve with llvm).

    Main inspiring languages:

    • C++ (oh yeah ! <3 <3 <3)
    • Rust (RAII and owner is the way to go)
    • Python (hight level API, cool syntax)
    • Ruby (cool syntax too)

    Here the Sugar repository

    It's really at the early stage. Here an example of what it can do already:

    Int a = 5
while a < 10
  echo a++

a = 5
while a < 10 {
        echo ++a
}

    This project may be renamed Silk.

    I like this new name because the pronouciation reminds C, and it follow the trend to use materials' names (like perl, ruby, ...).

    And I was working on this project while planning my Silk road trip ^^

    Chapi

    It's the project I'm working on currently.

    It's a device designed to control Black Magic Design devices.

    It's based on Raspberry Pi and buildroot for the device side, and on Qt on the configuration manager side, so the configuration manager could easily be ported on many platforms.

    As it will be used in professional world, it will soon have a more polish state.
